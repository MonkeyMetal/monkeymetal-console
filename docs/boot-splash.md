# Launcher 开机动画与音效设计说明书 (Boot Splash Design)

> **文档路径:** `docs/boot-splash.md`
> **版本:** v1.0.0
> **状态:** 设计阶段
> **目标:** 在系统启动器 `cart.lua` 启动时提供极具仪式感、科幻复古风格的开机动画和 8-bit 电子琶音音效，与底层硬件启动做无缝高质感过渡。

---

## 1. 整体构想

在掌机开机时，由于底层 C 语言已经播放了约 1.6 秒的静态 MonkeyMetal 图案及简单的三音符（C5-E5-G5）提示音。当 Lua 运行时接管并加载 Launcher（启动器）时，我们不直接显示冷冰冰的游戏列表，而是播放一段极具科幻和极客感的**系统装载过渡动画**，让玩家感受到系统正在以“超空间跃迁”和“自检”的形式真正启动。

该动画将完全基于 MonkeyMetal Console 的图形接口（`gfx`）和音频接口（`audio`）以**非阻塞方式（主循环更新驱动）**实现。

---

## 2. 动画视觉流程 (Visual Flow)

动画总时长设计为 **2.8秒**，包含以下三个连贯阶段：

```
🕒 时间线 (Seconds):
0.0s                      1.2s                               2.2s          2.8s
├──────────────────────────┼──────────────────────────────────┼──────────────┤
│ 阶段 1: 超空间跃迁 & 自检   │ 阶段 2: 标志汇聚与雷达扫描        │ 阶段 3: 就绪 │ 自动进入
│ - 星空粒子超速飞行 (Z轴加速) │ - 粒子减速，大字 LOGO 显现       │ - LOGO 闪烁  │ 游戏列表
│ - 滚动显示虚假 OS 装载日志    │ - 高亮扫描线自上而下扫过字形       │ - "READY!"   │ (page="games")
│ - 伴随磁盘读取的滴答声音效   │ - 播放震撼的 8-bit 上升大七琶音   │ - 界面淡出   │
```

### 2.1 阶段 1：超空间跃迁与系统自检 (0.0s - 1.2s)
- **3D 星空加速**：复用星空粒子数组，但粒子前进速度（`speed`）从平时的 `4` 飙升至 `24`，在屏幕上产生向外喷射的强烈穿梭感。
- **虚假自检日志 (Fake Boot Log)**：在屏幕左下方，以 8px 字体快速滚动打印自检信息：
  ```
  [OK] EXT_RAM_BSS INIT
  [OK] MOUNT SDCARD (/sdcard)
  [OK] ES8311 I2S CODEC ACTIVE
  [OK] BLE HOST ON (PAIRING_READY)
  [OK] LUA 5.4 VM READY
  [OK] SYSTEM ONLINE!
  ```
- **进度条**：屏幕下方有一个 1bpp Dither（棋盘格）填充的复古进度条，随着时间平滑从 0% 涨到 100%。
- **音效**：播放几声极短的高频滴答音（20ms），模拟老式软盘驱动器或磁带读取数据时的磁头寻道声。

### 2.2 阶段 2：LOGO 汇聚与雷达扫描 (1.2s - 2.2s)
- **星空减速**：粒子速度从 `24` 骤降到正常速 `3`，形成舒缓的星空背景。
- **LOGO 展现**：大字 `MONKEY METAL` 在屏幕中心显现。
- **雷达扫描线效果**：一根黑色高亮水平横线（扫描线）从文字顶部（Y=100）平滑扫过底部（Y=150）。扫描线扫过时，将文字的对应横行临时渲染为反色（白字黑底），呈现类似 CRT 显示器的光栅扫描高保真特效。
- **音效**：触发本次设计的高潮——一组璀璨上升的 **C大三/大七和弦 8-bit 电子琶音**（C5 - E5 - G5 - B5 - C6），声音清脆、宽广、科技感十足。

### 2.3 阶段 3：就绪与淡出 (2.2s - 2.8s)
- 显示 "**SYSTEM READY!**" 文本在底部闪烁。
- 粒子速度缓慢平复。
- 2.8 秒结束时，动画状态置为结束，平滑过渡到 `page = "games"` 游戏选择主界面，并重置闲置休眠计时。

## 3. 音效乐理设计 (Audio Synthesis)

在单片机上，由于我们的 bindings `audio.tone(freq, duration)` 每次只能发出单一波形，无法多轨合成，因此必须使用**快速琶音 (Arpeggio)** 来建立和弦感。这是经典 8-bit FC/NES 游戏机（如红白机）最常用的电子乐手法。

### 音谱与时间节点：

| 触发时间 (ms) | 音符 | 频率 (Hz) | 时长 (ms) | 作用 |
| :--- | :--- | :--- | :--- | :--- |
| **100** | - | 1200 | 15 | 模拟读取磁盘数据 Tick 1 |
| **300** | - | 1500 | 15 | 模拟读取磁盘数据 Tick 2 |
| **600** | - | 1000 | 15 | 模拟读取磁盘数据 Tick 3 |
| **800** | - | 1800 | 20 | 模拟读取磁盘数据 Tick 4 |
| **1200** (高潮启动) | **C5** | 523 | 80 | 琶音第 1 音（根音） |
| **1300** | **E5** | 659 | 80 | 琶音第 2 音（三音） |
| **1400** | **G5** | 784 | 80 | 琶音第 3 音（五音） |
| **1500** | **B5** | 988 | 100 | 琶音第 4 音（大七音，科技空灵感） |
| **1600** | **C6** | 1047 | 300 | 琶音最终完美终止高音（高八度根音） |

通过非阻塞的状态机时钟，这些音符会在最精准的时间被依次鸣响。

## 4. 程序实现架构

为了不让开机动画卡死（避免使用 `vTaskDelay` 或 `sleep`），动画将完全置于 Launcher 的 `page = "boot"` 状态中。

### 4.1 核心变量
- `page = "boot"`: 启动页面状态。
- `boot_start_ms`: 记录卡带 init 的系统时间戳，后续所有视觉与音效的 tick 都相对于这个值计算（`local elapsed = system.time_ms() - boot_start_ms`）。
- `boot_log_index`: 当前应该打印到哪一行自检日志。

### 4.2 非阻塞更新
```lua
-- 在 update() 中：
if page == "boot" then
    local elapsed = system.time_ms() - boot_start_ms
    
    -- 1. 更新星空粒子位置 (不同阶段速度不同)
    local star_speed = 4
    if elapsed < 1200 then
        star_speed = 24  -- 超光速跃迁
    else
        star_speed = 3   -- 正常漂浮
    end
    update_stars_with_speed(star_speed)
    
    -- 2. 非阻塞音效触发器 (仅触发一次)
    trigger_boot_audio(elapsed)
    
    -- 3. 过渡到主界面
    if elapsed >= 2800 then
        page = "games"
        last_input_ms = system.time_ms() -- 重置闲置计时，防止立刻进屏保
    end
    return
end
```

## 5. 性能与功耗优化

1. **画面无冗余绘制**：粒子保持在 `50` 个，由于进行了 Z 轴投影，超出边界的粒子会自动在后台随机重置位置，无需分配额外的 GC 内存。
2. **内存零开销**：自检日志文本定义为静态 local table，不产生运行期局部 String 拼接，防止触发 Lua VM 垃圾回收引起的掉帧卡顿。
3. **音效不锁死线程**：音效由底层 task 异步输出，Lua 脚本只负责在精准毫秒数发出控制指令，运行极为流畅，稳定跑满 33 FPS 硬件上限。
